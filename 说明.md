## 本地开发

先编译 native helper（只需一次，或 native 代码变更后）：

```powershell
cnpm run build:native:win
cnpm run dev
```

**为什么 `cnpm run dev` 第一次比较慢？**

这是 Electron + Vite 项目的正常现象，不是 cnpm 的问题：

1. **Vite 预构建依赖**（首次约 10–30 秒）：React、Pixi.js 等会先打包到 `node_modules/.vite`，第二次起会快很多。
2. **编译 Electron 主进程**（约 1–3 秒）：`electron/main.ts`、`preload.ts` 每次冷启动都会先编译一遍。
3. **启动 Electron**（约 1–2 秒）：相当于拉起一个完整的 Chromium + Node 进程。
4. **项目体积大**：编辑器、Pixi、视频处理、Whisper 字幕等模块较多。

只改前端时，保持 `dev` 进程不关，热更新会很快。只有改了 `electron/` 下代码才需要重启 dev。

## 打 Windows 安装包（一键脚本）

```powershell
cnpm run build:win:pack
```

脚本会自动设置国内镜像：

| 环境变量 | 用途 |
|----------|------|
| `ELECTRON_MIRROR` | Electron 二进制 |
| `ELECTRON_BUILDER_BINARIES_MIRROR` | NSIS、winCodeSign 等 electron-builder 工具 |
| `CAPTION_MODEL_MIRROR` | Whisper 字幕模型（ModelScope） |

跳过 native 编译（已编过、只改前端时）：

```powershell
.\scripts\build-win-pack.ps1 -SkipNative
```

安装包输出目录：`release\1.5.0\`（版本号随 package.json 变化）

## 常见问题

**本地已有 `electron` 依赖，还会下载吗？**

会。`node_modules/electron` 是 npm 包；打包时 electron-builder 仍要拉 **Electron 二进制 zip**（可缓存到 `%LOCALAPPDATA%\electron\Cache`）。另外还会拉 **NSIS** 等工具（走 `ELECTRON_BUILDER_BINARIES_MIRROR`）。

**报错 `Cannot create symbolic link : 客户端没有所需的特权`（winCodeSign）**

已在 `electron-builder.json5` 里设置 `signAndEditExecutable: false`，本地未签名打包可跳过 winCodeSign。
